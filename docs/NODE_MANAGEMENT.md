# Emby 节点管理

## 路径与展示名称

短路径是客户端连接的唯一标识，允许小写字母、数字、`_` 和 `-`，最长 32 位。例如别名 `charity`：

```text
https://relay.example.com/charity/<访问密钥>/
```

主代理机器启用双域名后，面板域名不再参与客户端地址生成。节点卡片“查看地址/复制”始终读取当前生效的本地代理域名；切换期间继续显示旧地址，只有 Nginx 与证书成功应用后才切换成新地址。旧代理域名随后不再接受节点请求。

展示名称只出现在管理面和可选的 `/gateway/` 公开节点页。公开页不会展示密钥或上游地址。

连接密码模式是默认值。创建节点时可以填写 8–64 位自定义连接密码（字母、数字、下划线或连字符），留空则安全随机生成。管理员登录后可在节点卡片点击“查看地址”，持续查看和复制完整客户端地址与连接密码。连接密码保存在 AES-256-GCM 加密的数据文件中，代理鉴权使用应用主密钥生成的 HMAC 摘要。公开短路径模式适合另有 VPN、mTLS 或严格 IP allowlist 的环境，不建议直接暴露公网。

从 `0.2.0` 更新的旧节点只保存了摘要，无法还原原密码。首次点击“查看地址”时，面板会询问是否更换密码；确认后旧客户端地址失效，新地址以后可以持续查看。

## 新手默认选择

- **连接密码**：开启，防止路径被扫描后直接盗用。
- **播放流量**：由代理全程接管。上游切换到网盘或 CDN 地址时，客户端拿到的仍是受保护的节点地址。
- **内网上游**：默认关闭；真实上游是 `192.168.x.x`、`10.x.x.x` 等自有地址时才开启。
- **TLS 证书验证**：保持开启；只在确认上游证书问题且短期排障时关闭。
- **公开节点页显示**：默认不显示，更私密；需要时通过 `/gateway/` 查看。
- **管理列表置顶**：只改变管理面排序，不影响代理功能。

## 多线路与熔断

主线路可用分号或换行分隔，最多 8 条。每条线路有独立的内存健康状态：

1. 请求在未熔断线路间轮转。
2. GET/HEAD 遇到连接错误、超时、502、503 或 504 时尝试下一条。
3. 连续失败两次后熔断 15 秒；持续失败时退避时间逐级增加，最长 5 分钟。
4. 当所有线路均熔断时允许探测，成功后立即恢复。

非幂等请求不会自动重放，因为登录和 API 写操作可能产生副作用。

## 前后端分离（推流网址反代）

有一类 Emby 服务器把**登录/API 与视频推流分属不同网址**（前后端分离）：推流网址以绝对 URL 的形式**写在 API 响应体里**（如 `PlaybackInfo` 的 `DirectStreamUrl`）。若不处理，客户端会直接读取该推流网址、绕过反代直连上游。

在节点编辑器的“推流网址反代”里开启并填入播放用的网址后：

- 文本类响应（JSON / XML / m3u8、DASH 清单等）里指向声明推流网址的绝对地址，会被改写为 `https://你的域名/<节点前缀>/.aegis-vod/<scheme>/<推流网址>/...`，客户端因此始终只用你的域名取流。登录网址填在“上游地址”即可，这里不用填。
- 对该路径的请求只会回源到**你声明过的推流网址**，其它主机一律拒绝（防 SSRF / 开放代理）；`ws`/`wss` 实时连接同样改写回本域名。
- **只改写文本/清单响应，媒体流零改动**：任何带 `Range` 的请求、`206`、`HEAD` 都直通转发，压缩协商也不改动，播放的启播、跑满带宽、拖拽 seek 与不开此功能时完全一致；仅个别需要改写的压缩文本响应会被就地解压后改写。

该功能只用于你**拥有或已获授权**的上游，用途是让多网址后端的流量收敛回你自己的域名，不用于隐藏客户端或绕过上游的访问控制。诊断里的“测试连接”会用与主回源相同的逻辑一并探测这些推流网址。

### 连接 Emby Proxy Toolbox 通用网关

上游地址必须保留 Toolbox 生成的完整基础路径，例如：

```text
https://gateway.example.com/工具箱密钥/http/origin.example.com:8096
```

HTTP 端口必须包含 `/http/`；默认格式会按 HTTPS 回源。AegisRelay 会在登录跳转时去除已经存在的 Toolbox 基础路径，再拼接节点短路径，避免路径重复导致 502。

从 `0.4.1` 起，播放请求会保留公网域名、HTTPS 协议、`Range` 和 `If-Range`。Toolbox 将目标改写到另一条网关路径时，AegisRelay 会把该目标封装为不可伪造的内部续传地址；客户端随后仍连接 AegisRelay，再由 AegisRelay 请求 Toolbox 的新目标路径，不会把基础路径重复拼接，也不会绕过当前代理机器。

如果可以登录和浏览媒体库、但点击播放立即失败，请先确认已更新到 `0.4.1` 或更高版本，然后重新创建容器：

```bash
sudo aegis-relay update
```

更新不会改变节点别名、连接密码或上游配置。

诊断探针会把连接失败、超时、TLS 问题及上游响应中途断开转换为面板内的故障结果，不应导致服务重启。如果点击“测试连接”时看到管理服务 502，请先更新到 `0.4.1`；更新后仍出现时再运行 `sudo aegis-relay logs` 查看容器状态。

## 播放流量接管

AegisRelay 不再提供客户端直连与服务端跟随之间的策略开关。任何合法的上游跳转都会先被封装为经过认证的内部地址，再返回给客户端；客户端发起后续请求时仍经过同一个节点路径和代理机器。

内部地址使用节点级密钥或应用密钥进行 AES-256-GCM 认证加密，不能被客户端改造成任意目标，因此不会把节点变成开放代理。跨域后会删除 `Authorization`、`Cookie` 和 `X-Emby-Token`，保留播放所需的 `Range` 与 `If-Range`，并继续执行 DNS/IP SSRF 检查。

## 获授权的客户端兼容配置

请求头配置支持以下通用字段：

- `User-Agent`
- `X-Emby-Client`
- `X-Emby-Device-Name`
- `X-Emby-Device-Id`

这些字段用于自有或获授权服务器的兼容测试，例如统一设备名称或修复旧服务的 UA 识别。系统不修改用户认证 Token，也不提供面向第三方限制的冒充模板。若上游明确禁止反代或某类客户端，应尊重其规则并关闭该节点。
